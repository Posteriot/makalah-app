# Style Guide Dokumentasi (ID + Technical EN)

## Bahasa
- Gunakan Bahasa Indonesia yang sederhana dan jelas.
- Boleh menyisipkan istilah teknis Inggris yang umum (contoh: component, props, state, hook, routing).
- Hindari kalimat bertele-tele; fokus ke fakta dari kode.

## Nada
- Netral, informatif, dan to-the-point.
- Jangan menyimpulkan hal yang tidak ada di kode.
- Jika perlu asumsi, tulis jelas sebagai asumsi.

## Struktur Umum
- Ringkasan 1 paragraf.
- Detail per file/fitur/komponen: poin-poin.
- Daftar file terkait.

## Konten Wajib
- Tujuan file/komponen.
- Data flow utama (props/state/hooks atau query/mutation Convex).
- Dependensi penting yang terlihat di kode.
- Constraint atau edge case yang eksplisit di kode.
